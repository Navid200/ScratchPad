# What's new about Dexcom G7 with respect to G6  
[xDrip](../../) >> [Features](../Features_page.md) >> [xDrip & Dexcom](../Dexcom_page.md) >> What's new about Dexcom G7 with respect to G6  
  
1- There are more fluctuations in readings specially during the first day of a sensor.  This is not related to xDrip.  
If you use AAPS, you can use one of existing smoothing options in AAPS to address this.  
  
2- A G6 session needs to be started from an app or receiver or pump.  A G7 session starts automatically when you insert the sensor.  
Therefore, there is no start sensor option for a G7 in xDrip.  
You need to stop an existing G6 session before you can start a new one.  
Since you don't need to start a G7 session, there is also no need to stop one.  That's why there is also no stop option for G7 in xDrip.  
  
3- You could use xDrip to connect to a G6 transmitter by just entering its serial number as transmitter ID even if there was another G6 transmitter in range.  
If you have a G7 that is active but not connected to xDrip or Dexcom app or receiver, and you want to use xDrip to connect to anotehr G7, you may have a hard time.  


  
4- You were advised to connect with a pump before connecting with xDrip to G6.  
It's not the same with G7.  If your pump connects directly to G7, you need to connect to G7 with xDrip first before the pump connects to it.  
  
5- G6 has only one mode of operation, transmitting once every 5 minutes.  
A G7 has a "rapid reconnect" mode as well in which it transmits once a minute.  It transitions to this mode if it has no connectivity for 15 consecutive minutes.
